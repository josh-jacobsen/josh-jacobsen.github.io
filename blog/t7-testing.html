Scope in Javascript

Javascript has two scopes: global and local. Scope refers to the set of variables that are able to be accessed: a global variable (a variable declared outside a function) can be accessed at any point. Local variables on the other hand (those declared inside a function) are local in scope; they can only be accessed by code within that same function.

The above provision about variables within functions being local only applies however if the var keyword is used. If a variable is declared inside of a function without using var, it is created as a global variable.

As a general rule, local variables are always prefered over global ones. Trying to hold to this guideline as much as possible is important especially when programs start getting bigger. Having a variable name that was reused (quite possible in larger programs) could have unexpected results that would take a significant amount of time to debug.
